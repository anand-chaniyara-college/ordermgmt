[BUG REPORT] AuthServiceImpl

Identified Bugs:

1. CRITICAL SECURITY FLAW: Revoked Token Reuse
   - Description: The `refreshToken` method fails to check if the refresh token has been marked as `revoked`. It allows a revoked token to generate a new access token.
   - Impact: If a token is stolen or a user logs out, the old refresh token can still be used to maintain access indefinitely.
   - Failing Test Case: `refreshToken_ShouldFailForRevokedToken` in `AuthServiceImpl_LoginoutRefressTest.java`
   - Reproduction Step: Call `refreshToken` with a token where `revoked = true`. It returns success instead of failure.

2. CRITICAL SECURITY FLAW: Inactive User Access
   - Description: The `refreshToken` method fails to check if the associated user account is active (`AppUser.isActive`).
   - Impact: A banned or deactivated user can continue to refresh their session and access the system as long as they have a valid refresh token.
   - Failing Test Case: `refreshToken_ShouldFailForInactiveUser` in `AuthServiceImpl_LoginoutRefressTest.java`
   - Reproduction Step: Call `refreshToken` for a user where `isActive = false`. It returns success instead of failure.
