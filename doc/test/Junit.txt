# JUnit Test Coverage Documentation

# 1. Registration (AuthServiceImpl_RegiTest.java)
**Focus:** User Registration
*   **Success Scenario (Customer):** Registers a new user with `CUSTOMER` role and creates a corresponding `Customer` profile.
*   **Success Scenario (Admin):** Registers a new user with `ADMIN` role *without* creating a `Customer` profile.
*   **Failure Scenario (Duplicate Email):** Prevents registration if email already exists.
*   **Failure Scenario (Invalid Role):** Prevents registration if the specified role name does not exist.
*   **Edge Case:** Throws `NullPointerException` when passed a `null` registration request.

# 2. Authentication & Session (AuthServiceImpl_LoginoutRefressTest.java)
**Focus:** Login, Token Refresh, and Logout
*   **Login Success:** Returns access & refresh tokens for valid credentials.
*   **Login Failure (User Not Found):** Handles non-existent email.
*   **Login Failure (Invalid Credentials):** Handles incorrect password.
*   **Login Failure (Inactive User):** Blocks login for inactive users.
*   **Login Edge Case:** Throws `NullPointerException` for null login request.
*   **Refresh Success:** Issues new access token for valid refresh token.
*   **Refresh Failure (Expired):** Rejects expired refresh tokens.
*   **Refresh Failure (Not Found):** Handles unknown refresh tokens.
*   **Refresh Edge Cases:**
    *   Throws `NullPointerException` for null refresh request.
    *   **BUG REPRODUCTION:** Checks if revoked tokens are rejected.
    *   **BUG REPRODUCTION:** Checks if inactive users are rejected during refresh.
*   **Logout Success:** Marks refresh token as revoked.
*   **Logout Idempotency:** Handles logout gracefully even if token is not found.

# 3. Customer Profile Management (CustomerServiceImplTest.java)
**Focus:** Customer Profile Access & Updates
*   **Get Profile Success:** Retrieves full profile when both user and customer record exist.
*   **Get Profile Partial:** Returns profile with just email if customer record is missing.
*   **Get Profile Failure:** Throws exception if user email not found.
*   **Update Profile Success (Existing):** Updates fields for existing customer.
*   **Update Profile Success (New):** Creates new customer record if one didn't exist.
*   **Update Profile Failure:** Throws exception if user email not found.
*   **Edge Cases:**
    *   `getCustomerProfile` with null email (throws RuntimeException).
    *   `updateCustomerProfile` with null email (throws RuntimeException).
    *   `updateCustomerProfile` with null DTO (throws NullPointerException).

# 4. Product Catalog (User View) (ProductServiceImplTest.java - PLANNED)
**Focus:** Available Products Listing
*   **Get Available Products Success:** Returns list of products with `availableStock > 0`.
*   **Price Mapping:** Correctly maps unit price from `PricingCatalog`.
*   **Filtering:** Excludes items with `zero` or `null` stock.
*   **Edge Case (Missing Price):** Handles missing price gracefully (defaults to 0).
*   **Edge Case (Empty Inventory):** Returns empty list if no products available.

# 5. Inventory Management (Admin View) (InventoryServiceImplTest.java - PLANNED)
**Focus:** Administrative CRUD Operations
*   **Add Item Success:** Successfully saves new inventory item.
*   **Add Item Duplicate:** Fails if trying to add an item with an existing ID.
*   **Add Item Null:** Throws Exception if DTO is null.
*   **Update Item Success:** Updates name, stock, and other fields for existing item.
*   **Update Item (Rename ID):** Verifies cascade logic (delete old + create new) when ID changes.
*   **Update Item Not Found:** Returns error or handles gracefully if item ID doesn't exist.
*   **Delete Item Success:** Removes existing item from inventory.
*   **Delete Item Not Found:** Returns error or handles gracefully if item ID doesn't exist.
# 6. Order Processing (OrderServiceImplTest.java - PLANNED)
**Focus:** Order Placement & Management
*   **Create Order Success:** Successfully places order, calculates total, and sets status to PENDING.
*   **Create Order Failure (Stock):** Fails if requested quantity > available stock.
*   **Create Order Failure (Validation):** Fails if customer profile incomplete or items missing.
*   **Cancel Order Success:** Cancels PENDING order.
*   **Cancel Order Failure:** Fails if order is already processed/shipped.
*   **Get Orders:** Retrieves order history filtering by customer email.
*   **Admin Status Update:** Verifies allowed state transitions (e.g., PENDING -> CONFIRMED).
*   **Inventory Update:** Verifies stock deduction on CONFIRMED and restoration on CANCELLED.

# 7. Pricing Management (AdminPriceServiceImplTest.java - PLANNED)
**Focus:** Price Updates & History
*   **Add Price Success:** Adds new price entry for an item.
*   **Add Price Failure:** Fails if Item ID is invalid.
*   **Update Price Success:** Updates the latest price for an item.
*   **Update Price Not Found:** Fails if no existing price record found.
*   **Get History:** Retrieves full price history for an item.
