Daily Useful Git Commands
=========================

1. Start Your Day (The "Breakfast" Habit)
-----------------------------------------
Always pull the latest changes from the main development branch before starting work.

# Switch to the main development branch (usually 'develop' or 'main')
git checkout <base-branch-name>

# Pull the latest code from the server
git pull origin <base-branch-name>

2. Create a New Branch
----------------------
Never work directly on `main` or `develop`. Create a specific branch for your task.
Naming Convention: feature/<TicketID>-<Short-Description>

# Create and switch to a new feature branch
git checkout -b feature/<ticket-id>-<feature-name>

# Examples:
# git checkout -b feature/KPT-24-Chatbot-Functionality
# git checkout -b bugfix/KPT-25-Login-Error

3. Check Your Status
--------------------
Use this frequently to see which files you have changed.

git status

4. Save Your Work (The "Dinner" Habit)
--------------------------------------
Commit your work every day before you leave.

Step 1: Stage your files
# Add all changed files
git add .

Step 2: Commit with a proper message
Format: <type> [optional scope]: <description>
Types: feat (new feature), fix (bug fix), docs (documentation), refactor (code cleanup).

git commit -m "<type>: <description>"

# Examples:
# git commit -m "feat: implement login api authentication"
# git commit -m "fix: resolve null pointer exception in order service"

5. Share Your Code
------------------
Push your local branch to the server so others can see it or review it.

# First time pushing this branch? Use this:
git push -u origin <your-branch-name>

# Afterwards, just use:
git push

6. Switching Context / Old Branch
----------------------------------
If you need to pause your current work and go back to an old branch or the main branch.

# 1. Save your current work first!
git add .
git commit -m "wip: saving work before switching"

# 2. Switch to the other branch
git checkout <other-branch-name>

# Example:
# git checkout main

7. Handling Hotfixes (Production Issues)
----------------------------------------
If a bug is found in production and needs an immediate fix.

# 1. Switch to master/main (production code)
git checkout master

# 2. Create a hotfix branch
git checkout -b hotfix/<ticket-id>-<issue-description>

# 3. Fix the bug, commit, and push
git add .
git commit -m "fix: <issue-description>"
git push -u origin hotfix/<ticket-id>-<issue-description>
